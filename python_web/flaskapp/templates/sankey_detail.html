What is Sankey?

Sankey is a specific type of diagram to visualize the flow of 1 specific section to another. The width of the line represents the quantity or frequency of the unit flow. Sankey diagrams are named after Irish Captain Matthew Henry Phineas Riall Sankey, who used this type of diagram in 1898 to show the energy efficiency of a steam engine. For further reference, you can visit: Sankey Wikipedia & Sankey Diagrams

What is Apriori Algorithm?

Apriori Algorithm is an algorithm for frequent item set mining and association rule learning over a set of data. It starts by identifying the frequent individual items and extending it to larger and larger item sets as long as those item sets appear sufficiently often in the dataset. Apriori algorithm was proposed by Agrawal and Srikant in 1994. For further reference, you can visit: Apriori Algorithm Wikipedia

What is support in Apriori Algorithm?

Support is the fundamental aspect in Apriori Algorithm. Support of an itemset X (suppX) is the proportion of transaction in the database in which the item X appears. Formula below: $$ supp(X) = { \text{No. of X in all transactions} \over \text{No. of Transactions}} $$ Increasing the threshold of support would eliminate arbitrary itemset and let us focus on the more frequent itemsets which may hold more significance.

What is confidence in Apriori Algorithm?

Confidence is the next aspect to look at in walking through the Apriori Algorithm. It is defined as follows: $$ conf(X \to Y) = { supp(X \cup Y) \over supp(X)} $$ Confidence in Apriori algorithm signifies the likelihood of item Y appearing when item X appears. It indicates the correlation between item Y and item X. It can also be interpreted as conditional probability. The probability of discovering item Y in a transaction set that already contains item X.

What does my event log contain?

Your event log contains list of services and events that are being logged to your computer from various sources be it the System itself or the Applications in your computer. Each of the services and events logged are associated with an event ID and event source. Each of these event IDs are associated with a single event type out of five available Event Types. We are particularly interested in the event IDs that are associated with Error event type as well as the number of their occurences.

How we are processing the event logs?

The raw dataset which is parsed from event logs contains a lot of information. In fact it is too many to be useful for our preliminary results. We first extract only the relevant columns such as event IDs, sources, and timing to be used in our analysis, strip the unnecessary text information, and sort it based on the timing. Then we calculate the frequency of the error events to visualize the top errors.

Visualizing the flow of event IDs require more complicated approach as we need to count the numbers of preceding events before the main error and collate it together to create a dataframe of event flows. Each rows of this dataframe represents a transaction set in Apriori's term. We then set out the necessary threshold for support to filter out arbitrary events and utilize Sankey Diagram to visualize only the most common ones.
