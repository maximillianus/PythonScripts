Welcome!

Welcome to the Event Log Web Application. This application serves to help users in analyzing the event logs in their computer by visualizing the statistics of that events properly. Kindly read the FAQs below to get started.

5W1H

What

What is this Analytics on Web?

Analytics on Web is a web application to help analyzing the event log in user's computer. This web application helps to visualize top errors, the rates they are happening, the flow of one error events to another, etc.

Who

Who is the appropriate audience for this web application?

This application is best suited to users who want to get better idea of the error events that are happening in their computer or workstation. This web application makes it easier for users to understand and analyze the root cause of certain errors in their computer

Why

Why would I want to use this web application?

There are a lot of useful information that can be grasped from event logs. These logs can help us to understand better errors and services that happened in our computer thus understanding event logs can create a better computing experience for people. Unfortunately going through thousands of lines of events are not efficient amidst of hectic schedule and heavy workloads. On top of that certain data-wrangling needs to be done in order to draw only the relevant useful information. This web application does all those chores and presents you beautifully what you need to see in a single button click.

When

When is it best to use this application?

This web application is best used when users encountered an error event (Example: Crash event) in their computer. It is best to take note the timing of that particular error event happened for cross-referencing purpose.

How

How is this web application being used?

Users will need to go to the shared folder location at \\10.111.240.124\Packaging$\Packages\1.New\Image\Aditya and copy the folder package eventlog_pkg to their own local drive. Then users will need to run the batchscript eventlog_extractToCSV.bat inside that folder to extract and convert their event logs to csv-format file. Users can then upload the csv file to this web application to visualize their error events. More detailed explanation can be read from Readme_LogParser.txt file in side the eventlog_pkg folder.
